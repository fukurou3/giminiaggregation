# コラム作成・管理ガイド

## 1. Firestoreでのコラム作成

### コレクション名
`columns`

### 必要なフィールド
```
{
  "title": "コラムタイトル",
  "slug": "url-friendly-slug",
  "author": "著者名",
  "excerpt": "概要文（150文字程度）",
  "coverImage": "https://example.com/image.jpg",  // オプション
  "category": "カテゴリ名",
  "body": "<p>HTML形式のコラム本文</p>",
  "createdAt": Timestamp,
  "updatedAt": Timestamp,
  "views": 0,
  "likes": 0,
  "isPublished": true,
  "featured": false  // トップページ表示用
}
```

### カテゴリ例
- 特集
- インタビュー  
- 技術
- 初心者向け
- 上級者向け
- ニュース

## 2. Firebase Consoleでの手動作成手順

1. Firebase Console → Firestore Database
2. 「コレクションを開始」→ `columns` 
3. 「ドキュメントを追加」
4. 上記フィールドを入力
5. 「保存」

### 注意事項
- `createdAt`, `updatedAt` は Firebase の Timestamp 型で設定
- `isPublished: true` にしないと表示されない
- `featured: true` にするとトップページに優先表示
- `body` フィールドはHTMLタグ使用可能

## 4. 表示確認方法

### トップページでの表示
- `featured: true` のコラムが優先表示される
- カルーセル形式で3記事まで表示
- 自動切り替え（3秒間隔）

### 表示される情報
- タイトル
- 概要文（excerpt）
- 著者名
- カテゴリ
- 作成日時
- 閲覧数・いいね数
- カバー画像（設定時）

## 5. 管理のベストプラクティス

### コラム企画
1. **ターゲット読者を明確化**
   - 初心者向け / 上級者向け
   - 特定ジャンル（ゲーム、アート、教育など）

2. **SEO対策**
   - titleは60文字以内
   - excerptに検索キーワードを含める
   - slugはアルファベットと数字のみ使用

3. **コンテンツの質**
   - 具体例・スクリーンショット付き
   - ステップバイステップの解説
   - 読者が実践できる内容

### 定期的なメンテナンス
- 古い情報の更新
- リンク切れのチェック
- 閲覧数・いいね数の分析
- 人気記事の特集化
